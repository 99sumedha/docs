    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-ios-runtime-changelog" class="anchor" href="#ios-runtime-changelog" aria-hidden="true"><span class="octicon octicon-link"></span></a>iOS Runtime Changelog</h1>

<h1>
<a id="user-content-0100-2015-april-17" class="anchor" href="#0100-2015-april-17" aria-hidden="true"><span class="octicon octicon-link"></span></a>0.10.0 (2015, April 17)</h1>

<p><a href="https://github.com/NativeScript/ios-runtime/issues?q=milestone%3A0.10.0">Milestone 0.10.0</a>, <a href="https://github.com/NativeScript/ios-runtime/releases/tag/v0.10.0">Release v0.10.0</a>.</p>

<h3>
<a id="user-content-fixed" class="anchor" href="#fixed" aria-hidden="true"><span class="octicon octicon-link"></span></a>Fixed</h3>

<ul>
<li>We will try to freeze the application on crash if there is a debugger attached so the debugger can be used to examine the errors.</li>
<li>When Objective-C exception is thrown from code called from JavaScript, it will be wrapped in JavaScript Error and reported to the debugger.</li>
<li>We have fixed the “tagged pointers“ bug on iPhone 5s with iOS7.0.</li>
<li>We have dramatically reduced the “tns-ios” package size by stripping the debug symbols from the NativeScript.framework.</li>
<li>We have fixed the project template to support properly the app-id provided from the CLI.</li>
<li>We’ve cleaned the package.tgz from some xcodebuild logs and the Chrome version of the inspector.</li>
<li>Made submodules public, fixed builds</li>
<li>
<em>tns_modules</em> are now expected in the app folder. We are backward compatible but will remove the compatibility in future.</li>
<li>We have updated the project template to use larger resolution</li>
<li>Promise reactions have too low a priority on the runloop</li>
</ul>

<h3>
<a id="user-content-new" class="anchor" href="#new" aria-hidden="true"><span class="octicon octicon-link"></span></a>New</h3>

<ul>
<li>JavaScript Date is implicitly converted to NSDate and vice versa.</li>
<li>JSON object and JS Map, when passed to native, are wrapped in NSDictionary. NSDictionaries do <strong>not</strong> behave as JSON objects when returned from native.</li>
</ul>
</article>
